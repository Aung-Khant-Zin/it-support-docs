# IT Incident Log Template

Use this template to log and track IT support incidents consistently.

---

## Incident Report

| Field | Details |
|-------|---------|
| **Incident ID** | INC-YYYYMMDD-001 |
| **Date Reported** | DD/MM/YYYY |
| **Time Reported** | HH:MM |
| **Reported By** | User's name / department |
| **Assigned To** | Technician name |
| **Priority** | 🔴 Critical / 🟡 High / 🟢 Medium / ⚪ Low |
| **Status** | Open / In Progress / Resolved / Escalated |

---

## Issue Description

> Briefly describe the issue as reported by the user.

**Example:** User is unable to connect to the company VPN from home. Error message: "Authentication failed."

---

## System Information

| Field | Details |
|-------|---------|
| **Device Name** | e.g. DESKTOP-ABC123 |
| **OS Version** | e.g. Windows 11 Pro 22H2 |
| **User Account** | e.g. john.doe@company.com |
| **Location** | e.g. Office / Remote |

---

## Troubleshooting Steps Taken

1. Verified internet connectivity on user's device — confirmed working
2. Checked VPN client version — outdated (v2.1), updated to v2.5
3. Re-entered VPN credentials — authentication successful after update
4. Tested VPN connection — stable, confirmed resolved

---

## Root Cause

> What caused the issue?

**Example:** Outdated VPN client was incompatible with recent server-side security update.

---

## Resolution

> What was done to fix it?

**Example:** Updated VPN client to latest version (v2.5). User confirmed able to connect successfully.

---

## Follow-Up Actions

- [ ] Notify team to push VPN client update to all remote users
- [ ] Update internal knowledge base with fix
- [ ] Monitor for recurrence over next 48 hours
